# Documentation Index

Repository documentation lives in `docs/`.

## Documents

- `docs/stack.md` - current application stack and architecture rules
- `docs/contributing.md` - contribution, git, commit, formatting, and review conventions
- `docs/testing.md` - unit testing strategy for the static-first application
- `docs/deployment.md` - Docker, Helm, GitHub Actions, and GitHub Pages deployment guidance
- `docs/github-workflows.md` - workflow intent and ownership

## Notes

- GitHub Pages is used for publishing repository documentation only.
- The application itself is prepared for container deployment, not static Pages hosting.
